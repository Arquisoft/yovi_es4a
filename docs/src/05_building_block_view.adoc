==  Vista de bloques

=== Visión general
image:03_2-ContextoTecnico.png["Diagrama correspondiente a la vista de bloque general", width=90%]
Vista general del sistema con tres microservicios independientes: webapp (interfaz), users (gestión de usuarios y multijugador) y gamey (motor del juego).

==== Descripción de los bloques

[cols="1,2,2", options="header"]
|===
| Componente | Responsabilidad | Interfaces

| *webapp*
| Presenta la interfaz gráfica al usuario. Permite jugar partidas en sus distintas modalidades (HvB, HvH local, multijugador online), registrarse e iniciar sesión.
| Consume la API REST de `users` para autenticación y estadísticas. Consume la API REST de `gamey` para crear partidas y aplicar movimientos. Se comunica mediante WebSockets (Socket.io) con `users` para el modo multijugador.

| *users*
| Gestiona el registro, login, estadísticas y historial de los usuarios. Coordina las salas multijugador en tiempo real. Expone una API REST documentada con Swagger.
| Se comunica con `MongoDB` para persistir datos de usuarios e historial de partidas. Expone endpoints REST consumidos por `webapp`. Gestiona conexiones WebSocket (Socket.io) para el multijugador. Métricas expuestas a Prometheus.

| *gamey*
| Motor del juego. Valida movimientos, mantiene el estado de las partidas activas en memoria (sesiones), gestiona las estrategias de bot y expone una API REST versionada.
| Expone una API REST bajo `/api/v1/` consumida por `webapp`. No tiene persistencia propia; el estado de cada partida se guarda en memoria durante su duración.

| *MongoDB*
| Almacena los datos de usuarios, estadísticas e historial de partidas.
| Solo accedida por `users` mediante lectura/escritura.
|===
